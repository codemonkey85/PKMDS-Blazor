When implementing an issue:

1. **Carefully review the issue description**: Make sure you understand the requirements and acceptance criteria before starting.
2. **Break down the issue into smaller tasks**: Identify the specific steps needed to complete the implementation.
3. **Write clean and maintainable code**: Follow coding standards and best practices to ensure your code is easy to read and maintain.
4. **Test your implementation**: Write unit tests and perform manual testing to verify that your implementation works as expected and meets the requirements.
5. **Document your code**: Add comments and documentation to explain your implementation and any important decisions made during development.
6. **Submit a pull request**: Once your implementation is complete and tested, submit a pull request to merge into main for review. Include a clear description of the changes made and any relevant information for reviewers.
7. **Update the PKHeX feature parity roadmap**: Ensure that the roadmap reflects the current state of features and any new additions or changes made during the implementation.

Tip: look in ../PKHeX (or ~/Code/PKHeX) for the PKHeX source code. You should reference the existing codebase to understand how similar features are implemented and to maintain consistency across the project.
